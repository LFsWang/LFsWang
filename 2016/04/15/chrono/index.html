C++11 std::chrono 時間和日期的操作介面

除了可以使用舊有的C函數之外，在C++11後標準提供了chrono，帶來了全新統一的處理方法。個人私下關注的一個專案，內部的計時器即是以此為基礎所製作出來的，在舊的C++中若想使用此功能的話，可以考慮使用c++ boost chrono，在boost的版本中，提供了其他未納入標準的功能(如IO處理)。

chrono對於時間的管理是十分有彈性的，允許使用不同的計時單位，並在各單位間做精準的單位換算。也被應用在C++11多線程調控；內置的高精準時鐘也解決了跨平台計時的問題，在STD的版本中，可搭配C函數來進行格式化輸出，入門難度筆者認為是中偏易，需特別小心在單位轉換上的限制與數值精準度的掌控。

std::chrono總共提供了3個主要的功能，下面逐一地介紹與詳解：

  • Duration 單位量長度的表示單位
  • Clocks 時鐘
  • Time point 用於表示點的類別

Duration

std::chrono::duration用於表示一個時程(time interval)，可於宣告時決定一個單位與標準單位的比例。提供了不同單位下的轉換介面，以便撰寫程式時可以使用更接近需求的單位，避免邏輯上的錯誤。

類別定義

名稱 型態 說明
Rep 基本算數型態 用於表示單位(ticks)數字的型態
Period 實數 std::ratio 每1單位Rep與標準單位的比例。在預設的時間處裡上是以秒(second)作為基本單位

我們可以利用Period這一個參數來完成單位轉換的功能，下面提供一個範例，用於轉換公尺(m)與公厘(mm)，並以公分(cm)作為標準單位。

程式碼使用了成員函數count()來輸出duration存有多少個單位(ticks)。輸出結果如下：

5m = 5000mm

成員列表

名稱 備註 描述
(constructor) 建構子 建構子
operator= 設定運算子
count() 回傳儲存了多少單位(ticks)
zero static 表示0單位量，數值等於Rep(0)
min static 表示最小能表示的單位量，數值等於std::numeric_limits<Rep>::lowest()
max static 表示最大能表示的單位量，數值等於std::numeric_limits<Rep>::max()
算數運算子 包含 一元+ 一元- + - * / % ++ -- += *= /= %=，與同等介面的外部函數，如果Rep可支援該運算的話。

duration最強大的功能在於不同類型間的duration運算時的轉換，因此使用時務必熟知建構子、設定運算子以及算術運算子的規則，接下來將會深入的講解這幾個的運作標準，若僅想概覽功能的讀者可以跳到：輔助

duration 建構子與設定運算子

duration建構子，除了有預設(default)的建構子，拷貝建構子，還具有下兩種形式。設定運算子是預設(default)的。

在上兩個建構子中，對於型態上有所限制，將兩個不同Rep的duration賦值時須遵守：

  1. Rep2可以被隱式(implicitly convertibl)轉換為Rep
  2. Rep是小數型態，或Rep2是非小數型態

第一點沒什麼太特別的問題，是為了確保兩者間是能運算的，第二點比較難以理解一點，簡而言之，你不能把整數轉換至小數的型態。因此duration<double> a = duration<int>(1);是不合法的。

對於不同的Period，也需滿足下面條件的其中一點：

  • Rep是小數型態
  • Rep2是非小數型態，且Period2是Period的整數倍。

所以上方的範例程式碼中，可以把meter設定給millimeter，但是如果換過來把millimeter設定給meter是不合法的，因為0.1不是100的整數倍。

若有使用的需求，要突破上面限制的話，應使用std::chrono::duration_cast來做型別轉換，功能類似於static_cast，內部實作上會盡可能避免除法以達到最佳精確度的要求。

統整上方規則，預設的基本型態數值設定：

你可以

  • 小單位(整數) = 大單位(整數)
  • 任意單位(小數) = 任意單位

你不可以

  • 大單位(整數) = 小單位(整數)
  • 任意單位(整數) = 任意單位(小數)

duration判斷小數使用的是std::chrono::treat_as_floating_point<T>，繼承自std::is_floating_point<T>。

duration判斷整除使用的是std::ratio_divide<Period2, period>::den == 1。

duration 算術運算子

如果你想用auto來自動判別duration計算完的結果，那這個章節非常重要，因為轉換的規則相當的複雜。這裡使用了特化的std::common_type<std::chrono::duration>來判別計算完的型態。計算完成的durationRep等於std::common_type<Rep,Rep2>，而Period等於Period與Period2的最大公因數(GCD)，計算方法在標準書有提供Note可以參考。

This can be computed by forming a ratio of the greatest common divisor of Period1::num and Period2::num and the least common multiple of Period1::den and Period2::den.

Working Draft Standard for Programming Language C++ 20.12.4.3 Specializations of common_type

下方用CD簡易表示std::common_type<A,B>。

  • L表示左邊的duration
  • R表示右邊的duration
  • d表示任意的duration
  • s表示一個表示單位(tick)的數值
運算子 返回型態 備註
L+R CD(CD(L).count() + CD(R).count())
L-R CD(CD(L).count() - CD(R).count())
s*d 或 d*s CD(CD(d).count() * s) duration只能乘以純量
L/s CD(CD(d).count() / s) duration除以純量還是duration
L/R CD(L).count() / CD(R).count() duration除以duration是純量
L%s CD(CD(L).count() % CD(R).count()) duration對純量取餘數是duration
L%R CD(CD(d).count() % s) duration對duration取餘數是duration

比較運算皆是先轉換為CD(L,R)後，再使用count()來進行比較。

duration 輔助方法

可查閱cpp reference duration，有預先定義milliseconds、seconds、minutes等等常用時間單位，並可使用1s，10min等Literals來表示seconds(1)與minutes(10)(C++14)。

輔助函數

名稱 功能
duration_cast() 類似static_cast，用於型別轉換，若小數轉換為整數，整數無法表示該小數時，為未定義行為(undefined behavior)
floor() C++17 incomplete implementation
ceil() C++17 incomplete implementation
round() C++17 incomplete implementation
abs() C++17 incomplete implementation

  • C++11 正式加入
  • C++14 加入一些literal representing(h,min,s,ms,us,ns)
  • C++17 加入一些算術函數

Clocks

Clocks題供了比C函數clock()解析度更高的時鐘，下面是標準規範的三種不同時鐘類別：

名稱 說明
std::chrono::system_clock 表示系統層級的時鐘
std::chrono::steady_clock 保證每次取值單調遞增的時鐘
std::chrono::high_resolution_clock 表示該系統上可使用解析度最高的時鐘

由於有些作業系統可以調整時間，因此可能發生取值時，後取的時間比先取的還來的早，因此steady_clock可能使用其他測量方法(例：計算開機後經過的時間)，來保證遞增的性質。這些類別取值(now())的回傳型態是time_point，將在下一章講解。

三個類別的介面一致，皆是靜態類別，使用時不需額外宣告，詳細構造可參考C++ concepts: TrivialClock

成員列表

名稱 備註 描述
rep type 算術型態 Clock用於存放標準單位的型態
period type std::ratio rep與1秒的比例 ( Clock一個rep是多少秒 )
duration type duration duration<rep,period>
time_point type time_point time_point<Clocks used>
is_steady static constexpr bool 表示該Clock是否保證now()取值單調遞增，其值視實作環境。steady_clock恆為true
now() static time_point 回傳目前時間點(time_point)
to_time_t(time_point) static time_t 由C++的time_point轉換C函數的時間類型
from_time_t(time_t) static time_point 由C函數的時間類型轉換為C++的time_point

實作備註

筆者翻閱了VC++2015的原始碼與GCC5.1的來比較，發現了下幾點，更新網路上過時的資訊，測試系統為Win10 x64

VC++2015

  • system_clock::now() 呼叫_Xtime_get_ticks對應到WinAPI：GetSystemTimePreciseAsFileTime/GetSystemTimeAsFileTime (前者需求Win8以上)
  • high_resolution_clock與steady_clock相等，對應到WinAPI：QueryPerformanceCounter

TDM GCC 5.1

原文件只有寫Map to C API，經過一點逆向後，發現呼叫了clock_gettime

  • high_resolution_clock與system_clock相等，呼叫clock_gettime(0,&v3)，更進一步追蹤，發現使用WinAPI : GetSystemTimeAsFileTime
  • steady_clock 呼叫clock_gettime(1,&v3)，使用WinAPI：QueryPerformanceCounter

比較

GetSystemTimePreciseAsFileTime具有號稱1微秒的精準度，是文件計時函數中最佳的精準度，但實際測試上卻是遠不及這個理想數據。網路上有許多這函數的比較與測試，在MSDN的說明頁提到GetSystemTimePreciseAsFileTime最佳狀況是用來精確度量日(time of day)或是用來同步時間，高精準測量應使用QueryPerformanceCounter。定義上high_resolution_clock要用系統最精確的時間，GCC用的是文件上的最佳，VC++用的實際上的最佳，導致用gcc編譯出的high_resolution_clock精準度比steady_clock低，實在是非常神奇。

Time point

用來表示時間點的類別，內部使用的型態與樣板參數傳入的Clocks一致，不再贅述，提供了一些算術上的函數可供使用。

標準定義

成員列表

名稱 備註 描述
(constructor) 建構子 建構子
time_since_epoch() duration 回傳該時間點與計時原點(可能是1970/1/1 00:00:00)間所經過的時間
加減運算子 可以與duration將加減，或是與另一個time_point運算
比較運算子 比較兩個time_point表示時間點先後順序
min 最小可表示的time_point，受Duration::min()影響
max 最大可表示的time_point，受Duration::max()影響

time_point 加減運算子

  • time_point 與 duration加減時，表示將該時間點位移duration的長度，回傳time_point，型態Duration是兩者的common_type
  • time_point 與 time_point 相減時，表示兩時間點間的長度，回傳duration，型態是兩者duration的common_type
