std::hash 標準雜湊介面

C++11 輔助函數物件 std::hash

std::hash是標頭functional的其中一員，提供了一致的雜湊介面，在非關聯性容器(unordered associative containers )中，擔任預設的雜湊函數。

hash設計概念

定義相當敖口，但是是C++實作上的重要條件，參閱自標準 17.6.3.4 Hash requirements。

一個物件T，若要成為C++的Hash物件須滿足下條件：

  1. T是一個函數物件(Function Object)
  2. T是CopyConstructible(*1)且Destructible(*3)的 (定義於標準 17.6.3.1)

且給定下條件

  • h，型態為T或const T的物件，且參數型態為Key
  • k，一個能轉換為Key或const Key型態的數值
  • u，型別為Key的L Value表達式

則下式子

表達式 回傳型態 規則
h(k) std::size_t 在該次程式運作時(duration of the program)(C++14)，輸出數值只被k影響(depends only on)。且對於相同的k，在執行時期(executed within a given execution of a program)(C++14)總是回傳相等的數值。且發生碰撞(*4)的機率應接近1.0/std::numeric_limits<std::size_t>::max()。
h(u) std::size_t u不會被更動修改

(*1) CopyConstructible須滿足MoveConstructible(*2)。當v是L Value時，執行T u=v，或T u(v)時，u與v會相等。
(*2) MoveConstructible指當v是R value時，執行T u=v，或T u(v)，u會與原來的v相等，執行完後v的值是不明確的。
(*3) Destructibl指的是物件T滿足解構時，~T()能正確回收所有資源，且不丟出例外。
(*4) 碰撞指 $a\neq b$，但$h(a)=h(b)$

解析

實作時，要滿足雜湊後的散列數值是平均分布的。而表達式被拆成兩個部分，第一個部分指的是所有的狀況下都要滿足的要求，而第二部分是當u是一個L value expr(ex. 一般變數)時額外要滿足的事項。

std::hash

標準已經提供我們一些基本型態的Hash可以使用，可參考下方的範例程式碼。

在實作上，hash可能使用trivial (identity) hash，也就是hash(T) == T這種，雖然該雜湊函數雖然滿足hash設計概念的要求，但是卻不是密碼雜湊函式(Cryptographic hash function)。

可以注意到C++14增加了一點敘述，只保證在同一個執行時期是雜湊相等的，這表示實作上可以在執行時期將雜湊加鹽(salted)，避免碰撞阻斷服務攻擊collision DoS attacks。

小心標準沒有提供C style string的雜湊，因此使用std::hash<const char*>時 指的是將指標的數值(address)進行雜湊，不是對字串(string)進行雜湊 。

範例

std::hash最廣泛的應用莫過於使用在非關聯性容器上了。要使用自定義的類別用在標準容器，比起關聯性容器只需要時做operator<，非關聯性容器需要實做operator==以及一個雜湊函數，相對複雜許多。

在第一個範例基本使用，簡單的印出雜湊過後的數值；第二個範例中，示範了侵入std中擴充hash功能的作法；第三個範例，則是定義一個滿足hash設計概念的自定義雜湊類別，來對一個vector進行雜湊。

使用自己的雜湊函數

標準並沒有納入hash_combine來合併兩個以上的雜湊數值，因此要將多個雜湊合併時必須自己撰寫。這個範例題供了一個boost使用的其中一個方法，中間一個特別的常數等於$Number Range / \phi$，$\phi$是黃金比例，數值等於$\frac{1+\sqrt{5}}{2}$

版本變更與註記

  • C++14後，標準也提供所有的列舉態使用hash。
  • C++17後，所有的整數型態(包含擴展的，如__int128之類的)皆可使用std::hash。
